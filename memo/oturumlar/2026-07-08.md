## Dilim 1: Oturum sürekliliği — TUI "önceki sohbete devam et" (BİTTİ ve testli)

Kullanıcının #1 önceliği. Kaldığımız yerden (DURUM.md'deki adım adım plan) devam edildi; plan
birebir uygulandı.

### Hedef
TUI'de her açılışta yeni sessionId üretiliyordu → geçmiş görmezden geliniyordu. Artık son sohbet
sürdürülebilir; yerel model (qwen) önceki konuşmanın bağlamını görür.

### Neden protokol/daemon değişmedi
- Geçmiş ZATEN REST'te: `GET /api/history/sessions?limit=N` + `/api/history/sessions/:id`
  (Faz 2, Bearer token'lı; `/api/health` dışı uçlar auth ister).
- Daemon sessionId'ye **REPLACE** semantiğiyle yazıyor (`store.ts`: session'ın mesajlarını sil +
  gelen tüm diziyi yeniden ekle). Yani eski sessionId'yi yeniden kullanıp tam mesaj dizisini
  yeniden göndermek yeterli → çiftleme yok. `chat.start`'a alan eklenmedi.

### Değişiklikler
1. **`cli/src/client/daemon-client.ts`** — `DaemonClient`'a REST geçmiş metotları:
   - `listSessions(limit=50)` → `HistorySessionSummary[]` (yeni→eski).
   - `sessionDetail(id)` → `HistorySessionDetailResponse | null` (404→null).
   - Ortak `getHistory(path)`: Bearer token + `AbortSignal.timeout`; 404→null, diğer hata fırlatır.
     Cevaplar shared zod şemalarıyla doğrulanır (kural 1). Port+token WS'ten zaten elde.
2. **`cli/src/tui/app.tsx`** — yeni `ChatFlow` bileşeni (dışa aktarıldı, test için):
   - `resumeModel` = önceki oturumun provider/model'ine uyan mevcut model (yoksa null).
   - `canResume` (kayıtlı sohbet + model mevcut) ise açılış durumu `resume-choice`, değilse `model-choice`.
   - Devam seçilince `loading` → `sessionDetail` → `ready` (model sabit + mesajlar tohum). system rolü
     filtrelenir (yalnız user/assistant geçmişe). Hata/detay-yok → `model-choice`'a düşer.
   - `runTui` açılışta `listSessions(1)` çeker (`.catch(() => [])` → geçmiş yoksa/başarısızsa sessiz,
     "devam et" gizlenir). `App`'e `lastSession` prop'u eklendi; eski `model` state'i `ChatFlow`'a taşındı.
3. **`cli/src/tui/resume-picker.tsx`** (YENİ) — "Yeni sohbet / Önceki sohbete devam et" seçici
   (mode/model picker deseni: ↑/↓+Enter; devam satırı provider/model · N mesaj · "başlık" gösterir).
4. **`cli/src/tui/chat.tsx`** — opsiyonel `initialSessionId?` + `initialHistory?` prop'ları; state/ref
   bunlarla tohumlanır (yoksa bugünkü davranış). `HistoryEntry` arayüzü dışa aktarıldı.

## Dilim 2: Birleşik sohbet-agent modu — ADR-012 + Dilim 2.1 (akış) BİTTİ

Kullanıcı "roadmap'i incele, devam edelim" dedi → sıradaki iş #2 (birleşik sohbet-agent modu).
Mimari ADR gerektiriyordu; kullanıcıya sorup **Seçenek A (konuşmalı motor) + akışlı (streamText)** onayı alındı.

### Tasarım (ADR-012 + PROTOKOL.md)
- İki yol (chat.start akışlı/araçsız vs agent.start araçlı/izinli/akışsız/tek-seferlik) **konuşmalı
  motorla** birleşir: konuşma = tamamlanınca `finish` yerine `awaiting_user`'a park olan çok-turlu
  agent koşusu; sonraki tur `agent.say`. Düz sohbet = araçsız "asistan" agent'ı. İzin kapısı/jail/
  araç döngüsü TEK yerde (engine) kalır → B (chat'e araç ekle: kapıyı çatallar) ve C (yeni converse.*:
  çoğaltma) reddedildi.
- PROTOKOL.md güncellendi (ADDITIVE, v1 korunur): `agent.delta` olayı, `awaiting_user` durumu,
  `agent.say` isteği, `agent.start.conversational?`. Dilimler: 2.1 akış → 2.2 çok-tur → 2.3 birleşik TUI.

### Dilim 2.1 — akış (streamText + agent.delta) — kod BİTTİ ve testli
- `shared/events.ts`: `AgentDeltaPayloadSchema {runId,text}` + map girdisi.
- `engine.ts`: `generateText`→`streamText`. Metin `for await (result.textStream)` ile tüketilip her
  parça `agent.delta` yayılır; sonra `await result.{response,usage,providerMetadata,text,toolCalls}`.
  Tool loop/izin/jail/finish/telemetri AYNI. **daemon.ts değişmedi** (bus otomatik yayar).
- Test mock'ları (`engine.test.ts` + `daemon-agent.test.ts`): `doGenerate`→`doStream`; scripted
  `turn()`→AI SDK v3 stream part'ları (`scriptToStream`). **Part şekilleri tahmin edilmedi**:
  `@ai-sdk/provider@4.0.1` `LanguageModelV3StreamPart` tiplerinden birebir okundu (stream-start,
  text-start/delta/end, tool-call{toolCallId,toolName,input}, finish{usage,finishReason}).
- `agent-run.tsx`: `agent.delta`→`streaming` state (green canlı render); araç başlayınca + run
  bitince temizlenir. +1 test.
- **Doğrulama:** build ✅ · test ✅ (engine 11 + daemon-agent 4 + agent-run delta; 209 geçer, welcome
  ortamsal) · lint ✅. Güvenlik-kritik kabul testleri (izin/jail/deny) streamText sonrası YEŞİL kaldı.
- **Canlı doğrulama kullanıcıya:** `symphony` → Agent → bir görev; asistan cevabı artık token-token akmalı.
